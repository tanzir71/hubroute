Security

Small enough to audit, strict enough to run on a real host.

Hardened runtime

Security headers, secure session settings, idle and absolute timeouts, generic error output, and server-side logging.

Safer data access

PDO prepared statements, whitelisted status filters, escaped output, and authorization checks around parcels, routes, hubs, agents, and CSV export.

Shared-hosting guardrails

The data directory receives Apache denial files, and setup notes recommend moving SQLite storage outside the web root whenever possible.

Checklist

Before this becomes a production parcel console.

  • Rotate every seeded password, especially the demo hub login shown on this page.
  • Keep `.env` private and move `DATA_DIR` outside `public_html` if your hosting plan allows it.
  • Confirm `data/hubroute.sqlite` is not downloadable after first run.
  • Enable HTTPS at the host or CDN and keep PHP on a supported 8.x release.
