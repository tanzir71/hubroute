Quick Start

Use the Vercel demo as an operator walkthrough.

1. Sign in as Dhaka North

Open hubroute.vercel.app, choose pickuphub@hubroute.local, and use hub1234.

2. Send a parcel onward

Open Parcels, use custody-flow filters, and send an eligible Dhaka parcel to Tejgaon Sortation Hub.

3. Switch hubs

Use Switch account, sign in as sortation@hubroute.local, then receive the incoming parcel.

4. Continue the path

Assign local sortation route/rider, record scan events, send to delivery, then inspect the redacted public tracker.

How It Works

Custody is explicit from booking to delivery attempt.

Intake

Create parcels with merchant, address, payment, weight, and service notes; route them into a pickup hub queue.

Sort and dispatch

Receive incoming parcels, clear old local assignment, sort by destination, and send onward through line-haul or last-mile runs.

Track and reconcile

Record events, delivery attempts, COD collection, and public tracking updates without exposing full PII.

Deployment

Landing and demo are intentionally split.

GitHub Pages

Serves index.html, docs.html, and repository documentation. It does not execute PHP.

Vercel demo

Serves the generated browser-only console from public/index.html. Data is local to each visitor's browser.

PHP source

hubroute.php remains the PHP 8 + SQLite app source for hosts that run PHP with PDO SQLite enabled.

Static bundle

Console source lives in src/demo/. Run npm run sync:demo to rebuild public/index.html and mirror vercel-demo/.

Security

Demo convenience and production credentials are separate concerns.

  • Do not reuse seeded demo emails or passwords in production.
  • Keep environment files and SQLite data outside public web paths when possible.
  • Use HTTPS, secure session settings, CSRF protection, prepared SQL, and escaped output for the PHP app.
  • Use role-based checks for parcel, route, rider, hub, and export operations.
Production Roadmap

Next steps after the browser walkthrough.

Product polish

Add mobile scanner ergonomics, batch assignment, printable labels, manifest export, SLA aging, exception queues, keyboard shortcuts, and denser table column controls.

Operational depth

Add service-area suggestions, COD settlement reconciliation, rider shift calendars, hub cut-off windows, route manifests, and delivery attempt reason taxonomy.

Scale architecture

Move from browser demo state to a server-backed API, indexed parcel/event storage, tenant-aware hub access, background jobs, audit logs, and observability.

Data evolution

Keep the custody event log append-only, add idempotent scan/event writes, model inter-hub bags/manifests, and harden the SQLite backup/restore path for larger operations.

Troubleshooting

Common demo and deployment checks.

  • If demo data looks stale, use Reset seed data in the sidebar or clear localStorage for hubroute.vercel.app.
  • If a parcel cannot be assigned, receive it at the active hub first; assignment is intentionally local to the current hub.
  • If a dropdown is long inside the operator console, type into it; all demo selects are searchable after JavaScript loads.
  • If the PHP app fails on a host, confirm PHP 8.1+, PDO SQLite, writable data directory, and private environment configuration.
